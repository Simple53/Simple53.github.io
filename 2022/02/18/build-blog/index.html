  • Hexo

  • 官网：Hexo

  • 教程：知乎

  • Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。即把用户的markdown文件，按照指定的主题解析成静态网页。

  • 4.博客主题

  • 4.1. config.yml

    网站的配置信息，您可以在此配置大部分的参数。 配置参数讲解

  • 4.2. package.json

    应用程序的信息，以及需要安装的模块信息。

  • 4.3. scaffolds

    模版文件夹。新建文章时，Hexo 会根据 scaffold 中的模板文件来建立新的文件。Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。也就是说，通过hexo命令每新建一个文章，都会包含指定模板文件中的内容。官网模板详述

  • 4.4. source

    资源文件夹是存放用户资源的地方，如markdown文章。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

    注意：除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。

  • 4.5. themes

    主题文件夹。Hexo 会根据主题来解析source目录中的markdown文件生成静态页面。官网主题详述
